# BRP-030 Runtime Renderer Harness

Date: 2026-05-23
Status: DONE

## Summary

Added ignored opt-in GPU harness tests in `backends/dear-imgui-bevy/src/render.rs`.

The harness initializes real Bevy/wgpu renderer resources via `initialize_renderer`, creates a
`PipelineCache`, constructs `GpuImage` values in `RenderAssets<GpuImage>`, and calls the production
Bevy image texture bind-group preparation path.

## Coverage

- Sampled `GpuImage` with `TextureUsages::TEXTURE_BINDING` registers a real
  `TextureBinding::Legacy` bind group.
- Missing `RenderAssets<GpuImage>` entries remove stale bind groups.
- Non-sampled `GpuImage` entries do not register bind groups.

package gates should report the harness as skipped rather than treating an early return as coverage.
