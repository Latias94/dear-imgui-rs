# DMV-060 Secondary Viewport Rendering

Date: 2026-05-23
Status: DONE

## Summary

- Added core platform-viewport draw-data snapshots so Bevy can inspect secondary Dear ImGui
  viewport draw data without retaining raw viewport pointers.
- Routed secondary viewport draw data through Bevy render extraction/preparation to the matching
  window render target.
- Enabled native requested `render,multi-viewport` status by setting render routing and full support
  true only when lifecycle, input feedback, and render routing are all available.
- Fixed shutdown cleanup by destroying Dear ImGui platform windows before clearing backend user-data
  and PlatformIO handlers.
- Preserved same-drain viewport lifecycle updates by merging `Create` plus initial
  `SetPos`/`SetSize`/`Show`/`SetTitle` commands before Bevy `Commands` flushes new windows.
