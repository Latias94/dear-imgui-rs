# 2026-05-25 - Closeout

## Summary

Closed the Bevy backend prelaunch refactor lane after landing the release-shaping renderer, texture,
input, documentation, and CI-stability changes.

## Decisions

- `render::ImguiOverlayCamera` is the explicit overlay camera marker. The backend keeps the
  highest-order camera fallback only when no active camera for a render target is marked.
- Camera viewport support uses Bevy's physical render pass viewport plus a logical projection rect
  derived from that camera viewport. Draw command scissors stay in framebuffer coordinates and are
  intersected with the camera viewport.
- Texture bind groups own sampler bindings. Registered Bevy images use `GpuImage::sampler`; managed
  ImGui textures keep standard linear/nearest sampler callbacks.
- BPR-060 is closed as target-dir pollution, not a backend test deadlock. Local release checks should
  use `CARGO_TARGET_DIR=target/bevy-backend-prelaunch` or another clean backend-specific target.

## Evidence

See `EVIDENCE_AND_GATES.md` for the fresh closeout gate table. The final gate set includes fmt,
native/wasm check, examples check, render and multi-viewport nextest, full-feature clippy, and
`git diff --check`.

## Residual Risk

- Pixel-level split-screen output is not covered by a GPU screenshot harness.
- wasm runtime clipboard/IME behavior remains host-dependent and documented as a support boundary.
