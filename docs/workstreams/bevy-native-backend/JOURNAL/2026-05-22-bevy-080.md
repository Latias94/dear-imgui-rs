# BEVY-080 — RenderApp Extraction

Implemented the first render-world extraction slice for the Bevy-native backend.

## Changes

- Added optional `bevy_render = "=0.19.0-rc.2"` to the backend `render` feature.
- Added `render::ImguiExtractedRenderFrame` as the render-world resource consumed by future renderer systems.
- Added `render::ImguiCameraTarget` to preserve active camera entity, order, and normalized render target.
- Installed extraction into Bevy `RenderApp` / `ExtractSchedule` using `bevy_render::Extract`.
- Cloned main-world `ImguiFrameOutput` snapshots across the extract boundary; raw ImGui draw data is not borrowed across worlds.
- Updated the render extraction test to use `ExtractPlugin` and assert that extracted state lives in `RenderApp`, not the main world.
