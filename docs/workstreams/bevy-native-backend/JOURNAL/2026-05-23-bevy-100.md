# BEVY-100 — Texture Feedback And Bevy Image Interop

Completed the texture-loop bridge needed by the editor viewport examples.

## Completed Slice

- Added `ImguiTextureFeedbackQueue` as a main-world resource shared with the Bevy render world.
- Applied queued renderer `TextureFeedback` on the main/UI thread before opening the next Dear ImGui frame.
- Added `ImguiBevyTextures` for stable, idempotent `Handle<Image>` registration as Dear ImGui `TextureId` values.
- Extracted Bevy image texture mappings into render world and resolved them through `RenderAssets<GpuImage>` during bind-group preparation.
- Kept managed texture draw bindings keyed by ImGui's managed texture identity while also reporting a renderer TexID alias through feedback.
- Updated README usage notes for texture feedback and Bevy image registration.
