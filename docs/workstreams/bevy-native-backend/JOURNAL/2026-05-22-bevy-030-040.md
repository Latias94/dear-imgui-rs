## BEVY-030 Notes

- `FrameSnapshot` carries draw metadata, draw lists, draw commands, and texture requests without retaining ImGui-owned pointers.
- Standard sampler callbacks are preserved as `DrawCmdSnapshot::SetSamplerLinear` and `DrawCmdSnapshot::SetSamplerNearest`.
- `TextureFeedback` can carry `TexID`, status, and renderer-owned backend user data for UI-thread application.
- Destroy feedback clears backend bindings through existing `TextureData::set_status(Destroyed)` semantics.
- Snapshot tests cover legacy texture binding, managed texture binding, create requests, update requests with tight upload rectangles, and draw-only snapshots.

## BEVY-040 Notes

- Added an extension integration test in `dear-imguizmo` because it can depend on the other extension crates as dev-dependencies without introducing runtime coupling.
- The test opens one `Context::begin_frame()` token, draws with ImPlot, ImNodes, imgui-node-editor, and ImGuizmo through the same `&Ui`, and closes with one `FrameToken::render_snapshot()`.
- No extension rewrite was required; Bevy should model plot/node/gizmo contexts as ECS resources and expose the same frame-scoped `&Ui` to user systems.
