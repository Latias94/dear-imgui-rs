# BEVY-070 Journal — ECS frame lifecycle scheduling

Date: 2026-05-22

## Scope

Implemented the main-world Bevy schedule slice that owns one primary Dear ImGui frame per Bevy app
update.

## Implementation Notes

- Added lifecycle schedules:
  - `ImguiBeginFrame`
  - `ImguiPrimaryContextPass`
  - `ImguiEndFrame`
- Inserted those schedules into the main schedule after `PreUpdate`, so BEVY-060 input mapping feeds
  IO before the frame opens.
- Added `ImguiFrameState` as a non-send resource that temporarily exposes the currently open `Ui`.
- Added `ImguiContexts` for user systems in `ImguiPrimaryContextPass`.
- Added `ImguiFrameOutput` for the latest completed frame index and `FrameSnapshot`.
- `ImguiEndFrame` renders once and snapshots draw data while raw ImGui pointers remain on the UI
  thread.
- Input-focused tests now run `PreUpdate` directly so they continue proving the input translation
  boundary without also consuming the full lifecycle schedules.
