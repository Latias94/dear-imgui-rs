## Gate Policy

Bevy `0.19.0-rc.2` requires Rust `1.95.0`, while this repository's root `rust-toolchain.toml` pins Rust `1.92.0`. BEVY-050 therefore uses dedicated `cargo +stable` gates for the Bevy crate and records that full workspace gates are not the authority for this task until the repository-wide MSRV changes or CI adds a dedicated Rust 1.95+ Bevy lane.

The skeleton now uses exact Bevy `0.19.0-rc.2` pin anchors for the app/ECS dependency tree after the intentional rc.2 target-train update. The earlier rc.1 skeleton showed that exact direct `bevy_app` / `bevy_ecs` deps alone can still allow transitive Bevy crates to drift to a newer release candidate. After the intentional rc.2 upgrade, explicit pin anchors keep the proof train coherently on rc.2 until the next target-train update.
