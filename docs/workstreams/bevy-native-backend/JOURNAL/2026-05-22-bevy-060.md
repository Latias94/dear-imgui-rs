# BEVY-060 Journal — Primary-window input mapping

Date: 2026-05-22

## Scope

Implemented the primary-window input slice for `backends/dear-imgui-bevy`.

The slice remains intentionally single-window/single-context. It queues Dear ImGui IO events from
Bevy messages but does not consume, delete, or stop Bevy input propagation.

## Implementation Notes

- Added `input::ImguiInputSystems` in `PreUpdate`.
- Added `ImguiInputState` for tracked focus, active touch, IME enabled state, and sticky
  key/button release.
- Added `ImguiInputCapture` as a Bevy resource that mirrors the latest Dear ImGui capture flags for
  game/editor policy decisions.
- Mapped primary-window:
  - window size and scale factor into display size / framebuffer scale;
  - mouse move, leave, button, and wheel messages;
  - physical keyboard keys, modifier aggregate keys, and text input;
  - `WindowFocused` and `KeyboardFocusLost` into focus events and stuck-input cleanup;
  - first active touch into touchscreen-source left mouse input;
  - `Ime::Commit` into text characters and explicit `Ime::Enabled` / `Ime::Disabled` into tracked
    IME state.
- Documented the input capture policy in `backends/dear-imgui-bevy/README.md`.
