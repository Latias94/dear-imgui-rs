# BEVY-090 — Renderer Pipeline Scaffold

Started the Bevy-native WGPU pipeline task with a narrow renderer-preparation slice.

## Completed First Slice

- Added local WGSL shader source and explicit `vs_main` / `fs_main` entry point constants.
- Added `ImguiGpuVertex` and `imgui_vertex_buffer_layout()` matching Dear ImGui draw vertices.
- Added `ImguiPreparedRenderFrame` and `ImguiPreparedDraw` as render-world preparation output.
- Flattened extracted `FrameSnapshot` draw lists into global vertices/indices plus per-camera draw commands.
- Converted ImGui clip rectangles into framebuffer-space scissor rectangles.
- Added `ImguiGpuBuffers`, which uploads prepared vertices/indices when `RenderDevice` and `RenderQueue` are available.
- Added `renderer_prepare_flattens_extracted_snapshot_for_pipeline_consumption`.
- Added embedded shader asset registration, `ImguiRenderPipeline`, `ImguiPipelineGpuResources`, `ImguiTextureBindGroups`, and `ImguiQueuedPipelines`.
- Added Core2d/Core3d overlay pass insertion. The pass currently draws only commands whose texture binding has a registered bind group.
- Added `renderer_pipeline_resources_and_descriptors_are_installed`.
