# BBP-040 Editor Product Slice

Date: 2026-05-23

## Summary

Upgraded `backends/dear-imgui-bevy/examples/editor/editor_shell.rs` from static editor panels into
a real ECS-backed editor slice:

- the sample scene now has a named `Scene Root` entity with child entities;
- the Hierarchy panel renders Bevy `Children` recursively and selects real `Entity` IDs;
- the Inspector panel syncs selected entity data into editable buffers;
- selected `Name` and `Transform` changes are written back through Bevy `Commands`;
- animated entities expose their `EditorSceneObject` motion data in the inspector.

The work remains inside the backend example surface. No reusable editor crate was introduced.

## Verification

- `cargo +stable fmt --all --check` - PASS.
- `cargo +stable check -p dear-imgui-bevy --features render --example editor_shell` - PASS.
- `cargo +stable test -p dear-imgui-bevy --features render --example editor_shell` - PASS, 2
  tests run.

## Notes

This slice intentionally avoids broad editor-application abstractions. If the editor surface grows
past example scope, split a separate editor workstream instead of hiding product state inside the
backend crate.
