## 1. 项目概述

本项目提供了现代化、类型安全的 Dear ImGui Rust 绑定，采用分层架构设计，支持多种渲染后端和平台。

### 架构设计

#### 分层架构
- **dear-imgui-sys**: FFI 层，直接绑定 C++ 代码 ✅ **完全工作**
- **dear-imgui-core**: 核心抽象层，提供内存安全的基础 API ✅ **完全工作**
- **dear-imgui**: 高级 API 层，提供易用的 Rust 接口 ✅ **完全工作**
- **backends/**: 独立的后端 crates
  - `dear-imgui-wgpu`: WGPU 渲染后端 ✅ **生产就绪**
  - `dear-imgui-winit`: Winit 平台后端 ✅ **生产就绪**
  - 计划中: `dear-imgui-opengl`, `dear-imgui-vulkan` 等

#### 设计原则
- 类型安全：利用 Rust 的类型系统防止常见错误 ✅ **完全实现**
- 零成本抽象：不引入额外的运行时开销 ✅ **验证通过**
- 模块化：每个后端独立，按需使用 ✅ **完全实现**
- 兼容性：支持 Dear ImGui 的所有原生功能 ✅ **基础功能完整**
- 高级 API：避免直接使用 unsafe 代码，提供类型安全的接口 ✅ **完全实现**
